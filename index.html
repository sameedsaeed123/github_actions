01 — What is it?

Your repo.
Your pipeline.

GitHub Actions is a continuous integration and continuous delivery (CI/CD) platform built directly into GitHub. It allows you to automate your build, test, and deployment pipeline from within your repository using simple YAML files.

No external CI server. No third-party integrations required. Your automation lives alongside your code.

⚡

Event-Driven

Trigger workflows on push, pull request, schedule, or any GitHub event.

📦

Reusable Actions

Use thousands of community-built actions from the GitHub Marketplace.

🌍

Any Platform

Run on Linux, macOS, Windows — or host your own self-hosted runners.

🔐

Secure Secrets

Manage API keys and tokens safely with encrypted secret storage.

02 — Core Concepts

The Building Blocks

01

Workflow

A configurable automated process defined in a .yml file inside .github/workflows/. A repo can have multiple workflows.

02

Event

A specific activity that triggers a workflow — like a push, pull_request, release, schedule, or manual workflow_dispatch.

03

Job

A set of steps that run on the same runner. Jobs run in parallel by default, but can be chained with needs:.

04

Step

An individual task inside a job. Steps can run shell commands or invoke Actions. They share the same filesystem within a job.

05

Action

A reusable unit of code — either from the Marketplace, your repo, or inline. The atoms that compose powerful workflows.

06

Runner

The server that executes your jobs. GitHub provides hosted runners; you can also self-host for custom environments.
